tabula-py

La simplicité pour les tableaux avec bordures

tabula-py est le wrapper Python de Tabula-Java. Il excelle quand les factures possèdent des lignes et bordures claires. Idéal pour des scripts rapides, mais nécessite un environnement Java.

Points forts• API ultra simple (2‑3 lignes)
• Retour direct DataFrame pandas
• Export CSV / JSON natif
• Parfait pour documents structurés
Limites• Dépendance Java (JRE obligatoire)
• Moins bon sans bordures (moins de 65% de précision)
• Pas de debug visuel
• Mémoire élevée (~120 Mo)

Tutoriel tabula-py

# 1. Installer Java (Ubuntu: sudo apt install default-jre)
# 2. pip install tabula-py pandas

import tabula

# Extraction simple
dfs = tabula.read_pdf("facture.pdf", pages="1")
print(dfs[0])

# Plusieurs tableaux + zone personnalisée
tables = tabula.read_pdf("facture.pdf", pages="1",
                         area=[100,50,500,800],
                         multiple_tables=True)

# Export direct CSV
tabula.convert_into("facture.pdf", "extrait.csv", output_format="csv", pages="all")

Conseil : utilisez cet outil pour des factures standardisées (comptabilité classique). Testez d'abord avec des paramètres par défaut.

camelot

Précision extrême, modes lattice et stream

camelot est une bibliothèque Python pure avec un mode lattice (bordures) >97% de précision, et un mode stream pour les tableaux sans grille visible. Fournit un rapport de qualité et des outils de visualisation.

Points forts• Précision record lattice (97%+)
• Pas de dépendance externe
• parsing_report + debug image
• Idéal pour grilles comptables
Limites• Mode stream instable (50-70% de précision)
• Documentation dense
• Performances réduites sur très gros PDF (+15 pages)

Tutoriel camelot

# pip install camelot-py[cv] pandas

import camelot

# Mode lattice (bordures) : recommandé pour factures classiques
tables = camelot.read_pdf("facture.pdf", pages="1", flavor="lattice")
print(f"Tableaux trouvés : {tables.n}")
print(tables[0].df)

# Exporter et rapport
tables[0].to_csv("lignes_facture.csv")
print(tables[0].parsing_report)   # précision + confiance

# Mode stream (sans bordures)
stream_tables = camelot.read_pdf("facture_moderne.pdf", pages="1",
                                 flavor="stream", edge_tol=500)

# Visualisation debug (nécessite ghostscript)
tables[0].to_image().show()

Cas d'usage : factures avec lignes de tableau bien tracées. Le mode lattice détecte chaque cellule avec une grande fidélité.

pdfplumber

Contrôle granulaire, sans bordures et debug visuel

pdfplumber donne un accès bas niveau aux caractères, lignes et boîtes. Idéal pour les factures modernes sans lignes apparentes, mais nécessite du paramétrage par fournisseur. Rapide et léger.

Points forts• Contrôle pixel (tolérances, stratégies)
• Debug visuel (image annotée)
• Gère cellules fusionnées, multilignes
• Très rapide (~1.5 pages/s) & léger (mémoire réduite)
Limites• Nécessite un réglage par format de fournisseur
• Courbe d'apprentissage élevée
• Extraction tableau moins automatique que camelot

Tutoriel pdfplumber

# pip install pdfplumber pandas

import pdfplumber

with pdfplumber.open("facture.pdf") as pdf:
    page = pdf.pages[0]
    # Extraction automatique (brute)
    table = page.extract_table()
    # Customisation fine
    table_custom = page.extract_table({
        "vertical_strategy": "text",
        "horizontal_strategy": "text",
        "snap_tolerance": 3,
        "join_tolerance": 5,
        "edge_min_length": 3
    })
    import pandas as pd
    df = pd.DataFrame(table_custom[1:], columns=table_custom[0])
    print(df)

# Debug visuel : génère une image avec les zones détectées
with pdfplumber.open("facture_sans_bordure.pdf") as pdf:
    page = pdf.pages[0]
    tables = page.find_tables()
    im = page.to_image(resolution=200)
    for t in tables:
        im = im.draw_rect(t.bbox)
    im.show()   # ou im.save("debug.png")

Astuce : Associez extract_words() pour analyser les positions verticales et définir vos propres frontières de tableau.

Approche hybride pour la production

def extraire_facture_complete(pdf_path):
    import camelot, pdfplumber, pandas as pd
    result = {"texte": "", "tableau": None}
    # Contexte texte avec pdfplumber
    with pdfplumber.open(pdf_path) as pdf:
        result["texte"] = pdf.pages[0].extract_text()
    # Tableau : essayer camelot lattice en priorité, fallback pdfplumber
    try:
        tables = camelot.read_pdf(pdf_path, pages="1", flavor="lattice")
        if tables.n > 0:
            result["tableau"] = tables[0].df
            return result
    except:
        pass
    # fallback pdfplumber extraction
    with pdfplumber.open(pdf_path) as pdf:
        t = pdf.pages[0].extract_table()
        if t:
            result["tableau"] = pd.DataFrame(t[1:], columns=t[0])
    return result

Exemple pratique : extraction de lignes de facture

Supposons un fichier nommé invoice_123.pdf avec un tableau (produit, quantité, prix unitaire, montant). Voici comment les trois bibliothèques s'en sortent :

tabula-py :
import tabula
df = tabula.read_pdf("invoice_123.pdf", pages="1")[0]
# Colonnes automatiquement détectées
camelot :
import camelot
tables = camelot.read_pdf("invoice_123.pdf", pages="1", flavor="lattice")
data = tables[0].df
data.to_excel("facture.xlsx")
pdfplumber :
import pdfplumber, pandas as pd
with pdfplumber.open("invoice_123.pdf") as pdf:
    table = pdf.pages[0].extract_table({
        "vertical_strategy": "lines", 
        "horizontal_strategy": "lines"
    })
    df = pd.DataFrame(table[1:], columns=table[0])

Résultat : pdfplumber est le plus robuste sur les polices variables, camelot le plus fidèle aux bordures, tabula-py le plus rapide à coder.
